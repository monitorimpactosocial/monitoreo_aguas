Metodologia del sistema

Como se construye, valida e interpreta el monitoreo

Esta vista documenta la logica completa del tablero: fuentes, carga, clasificacion territorial, comparabilidad, calculos estadisticos, limites de interpretacion y gobierno de datos.

1. Proposito y alcance

El sistema organiza informacion de calidad de agua para seguimiento ambiental, control operativo y lectura ejecutiva. No convierte automaticamente una diferencia numerica en impacto causal: primero revisa cobertura, punto, temporada, n, desviacion, fuente y comparabilidad.

2. Fuentes de informacion

La base publicada usa una capa JSON normalizada para que el tablero cargue rapido. El backend Apps Script recibe nuevas mediciones en el libro en linea y mantiene hojas operativas: `BD_PRINCIPAL`, `PUNTOS`, `USUARIOS_APP`, `SESIONES_APP` y `AUDITORIA_CAMBIOS`.

3. Puntos y clasificacion territorial

Los puntos se agrupan por componente, medio, cauce o sistema, y tipo de punto. La regla operativa del proyecto es rojo igual entrada, amarillo igual intermedio y verde igual salida. Rio Paraguay se lee con entrada, medio y salida; arroyos y pozos se mantienen como sistemas separados.

4. Carga de datos

Cada carga representa una muestra o campana para un punto, ano y temporada. Los parametros no medidos quedan vacios; no se completan como cero. Los textos de laboratorio como `<0,01`, `ND` o `S/D` se preservan para trazabilidad.

5. Seguridad y auditoria

El acceso es obligatorio. El primer usuario registrado queda como administrador y los siguientes como responsables. Las contrasenas se guardan con hash salteado, las sesiones vencen y cada carga registra usuario, fecha, accion, punto, periodo, resumen y origen.

6. Series e indicadores

Una serie combina componente, medio, cauce, punto, parametro y periodo. La app calcula cobertura, cantidad de puntos, cantidad de parametros, registros comparables, alertas por desviacion y cambios porcentuales frente a linea de base cuando existe soporte suficiente.

7. Lineas de base

Las referencias principales son 2021 como linea historica y 2023 como referencia reciente cuando existen datos del mismo punto y parametro. La diferencia porcentual se interpreta junto con n, desviacion estandar y continuidad del punto.

8. Comparabilidad

Un punto puede tener datos en muchos anos y aun asi no ser estrictamente comparable si cambio de ubicacion, si el n es bajo o si no existe el mismo parametro. El filtro de comparables reduce lecturas enganosas y prioriza puntos con soporte minimo.

9. Estadistica para muestras pequenas

El tablero distingue datos referenciales de evidencia comparable. Cuando hay muestras pequenas, se priorizan pruebas no parametricas o lectura cautelosa. Shapiro-Wilk evalua normalidad, ANOVA compara medias si corresponde y Kruskal-Wallis compara grupos sin exigir normalidad.

10. Alertas por desviacion

La alerta por desviacion usa el cambio frente a base dividido por la desviacion estandar disponible. Sirve para priorizar revision tecnica; no prueba por si sola una causa. El umbral del panel lateral ajusta la sensibilidad.

11. Lectura del mapa

El mapa permite entender posicion, cauce, tipo de punto y gradientes entrada-medio-salida. Al hacer clic en un punto se filtran todas las vistas para evitar mezclar sistemas no comparables.

12. Como interpretar resultados

La lectura correcta sigue este orden: confirmar fuente, revisar filtros, verificar n y comparabilidad, mirar tendencia, contrastar alertas, revisar tabla tecnica y recien despues redactar una conclusion ambiental.
